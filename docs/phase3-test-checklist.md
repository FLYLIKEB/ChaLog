# Phase 3 모바일 UI 개선 테스트 체크리스트

## 📋 테스트 항목

### 1. 차 선택 드롭다운 키보드 대응 테스트
**목적**: 모바일에서 키보드가 열렸을 때 드롭다운이 키보드 위에 표시되는지 확인

**테스트 방법**:
- [ ] 모바일 기기 또는 브라우저 개발자 도구 모바일 모드 사용
- [ ] `/note/new` 페이지 접속
- [ ] "차 선택" 입력 필드 클릭
- [ ] 키보드가 열리면 드롭다운이 키보드 위에 표시되는지 확인
- [ ] 드롭다운 항목을 터치할 수 있는지 확인 (최소 44x44px 터치 영역)
- [ ] `/note/:id/edit` 페이지에서도 동일하게 테스트

**예상 결과**:
- ✅ 드롭다운이 `fixed` 포지셔닝으로 키보드 위에 표시됨
- ✅ 드롭다운 항목이 키보드에 가려지지 않음
- ✅ 각 항목의 최소 터치 영역이 44x44px 이상

---

### 2. 이미지 Lazy Loading 테스트
**목적**: 스크롤 시 이미지가 지연 로딩되는지 확인

**테스트 방법**:
- [ ] 홈 페이지(`/`) 접속
- [ ] 여러 노트가 있는 경우 스크롤 다운
- [ ] 브라우저 개발자 도구 → Network 탭 열기
- [ ] 이미지가 뷰포트에 들어올 때만 로드되는지 확인
- [ ] NoteCard, NoteDetail, ImageUploader의 이미지 모두 테스트

**예상 결과**:
- ✅ 이미지에 `loading="lazy"` 속성이 있음
- ✅ 뷰포트 밖의 이미지는 초기에 로드되지 않음
- ✅ 스크롤 시 이미지가 순차적으로 로드됨

**확인 방법**:
```javascript
// 브라우저 콘솔에서 실행
document.querySelectorAll('img').forEach(img => {
  console.log(img.src, img.loading);
});
```

### 3. 텍스트 가독성 개선 테스트
**목적**: 텍스트 크기와 간격이 모바일에서 읽기 쉬운지 확인

**테스트 방법**:
- [ ] BottomNav 확인
  - [ ] 하단 네비게이션의 텍스트가 `font-medium`으로 표시되는지 확인
  - [ ] 텍스트 크기가 적절한지 확인
- [ ] NoteCard 태그 확인
  - [ ] 태그의 크기가 `text-xs sm:text-sm`로 반응형인지 확인
  - [ ] 태그 간격이 `gap-1.5`로 적절한지 확인
  - [ ] 태그 패딩이 `px-2.5 py-1`로 충분한지 확인
  - [ ] 태그에 `font-medium`이 적용되었는지 확인

**예상 결과**:
- ✅ BottomNav 텍스트가 더 명확하게 보임
- ✅ NoteCard 태그가 모바일에서도 읽기 쉬움
- ✅ 태그 간격이 적절하여 터치하기 쉬움

### 4. 성능 최적화 테스트
**목적**: React.memo 적용으로 불필요한 리렌더링이 방지되는지 확인

**테스트 방법**:
- [ ] React DevTools Profiler 사용
- [ ] 홈 페이지에서 노트 목록 렌더링
- [ ] 다른 노트에 좋아요/북마크 클릭 시 현재 노트가 리렌더링되지 않는지 확인
- [ ] 스크롤 시 성능이 개선되었는지 확인

**예상 결과**:
- ✅ NoteCard가 memo로 감싸져 있음
- ✅ props가 변경되지 않으면 리렌더링되지 않음
- ✅ 스크롤 성능이 개선됨

**확인 방법**:
```javascript
// React DevTools에서 확인
// 1. Components 탭에서 NoteCard 선택
// 2. "Why did this render?" 확인
// 3. props가 변경되지 않으면 "props did not change" 표시
```

### 5. 모바일 환경 전체 테스트
**목적**: 모든 Phase 3 변경사항이 모바일에서 정상 작동하는지 확인

**테스트 방법**:
- [ ] 실제 모바일 기기 또는 브라우저 개발자 도구 모바일 모드 사용
- [ ] 주요 페이지 순회:
  - [ ] 홈 페이지 (`/`)
  - [ ] 검색 페이지 (`/search`)
  - [ ] 새 노트 작성 (`/note/new`)
  - [ ] 노트 상세 (`/note/:id`)
  - [ ] 노트 수정 (`/note/:id/edit`)
  - [ ] 사용자 프로필 (`/user/:id`)
- [ ] 각 페이지에서 다음 확인:
  - [ ] 터치 영역이 최소 44x44px인지
  - [ ] 레이아웃이 깨지지 않는지
  - [ ] 키보드가 열렸을 때 UI가 가려지지 않는지
  - [ ] 이미지가 정상적으로 로드되는지
  - [ ] 텍스트가 읽기 쉬운지

**예상 결과**:
- ✅ 모든 페이지가 모바일에서 정상 작동
- ✅ 터치 영역이 충분히 큼
- ✅ 레이아웃이 반응형으로 작동
- ✅ 키보드와의 충돌 없음
